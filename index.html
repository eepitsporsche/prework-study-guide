HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta element defines metadata such the character set, description, keywords, author, and viewport.
  • "Child elements" are nested within a "parent element".
  • An "attribute" defines additional information about an element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class attributes allow you to share a CSS rule to any specific element by assigning the rule to a class attribute with a class selctor.
  • Inline CSS styles an element directly into HTML. It applies to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. It is useful for quick, permanent changes and ust be edited individually whenever you make a design change.
  • External CSS style sheets allow developers to store all their CSS rules in a separate file. It uses a link element placed in the HTML's head.

Git

  • "git status" checks what branch we are currently on.
  • "git checkout -b feature branch name" creates a new branch and switches to it.
  • "git add -A" adds code from the main branch to the current working branch.
  • "git pull origin main" checks if the local branch is synced with the base branch on Github.
  • "git push origin feature branch name" pushes changes to the remote branch on Github.

Javascript

  • A variable is a named conatiner that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are a way of packaging functionality you wish to reuse.
  • Conditionals are code structures to test if an expression returns true or not.
  • Events are code structures that listen for activity (such as clicks) in a browser and run code in response.
  • An array is a single variable used to hold a group of data.
